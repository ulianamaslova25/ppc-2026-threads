## 1. Контекст
Разреженные матрицы, в которых большая часть элементов — нули, неэффективно хранить в виде плотных массивов. Использование формата CRS (Compressed Row Storage) позволяет экономить память и ускорять вычисления. Последовательная версия необходима как эталон корректности и baseline для оценки ускорения параллельных версий.

## 2. Постановка задачи
- **Вход:** Две матрицы A (размера M x K) и B (размера K x N) в формате CRS.
- **Выход:** Матрица C = A * B в формате CRS.
- **Ограничения:** Число столбцов A должно совпадать с числом строк B. Векторы `row_ptr` должны иметь размер `rows + 1`.
- **Крайние случаи:** Умножение на вектор-строку, пустые матрицы, полностью плотные матрицы.

## 3. Базовый алгоритм
Использован алгоритм Густавсона:
1. Для каждой строки $i$ матрицы A:
   - Подготавливается временный плотный аккумулятор (`temp_row_`) и массив меток (`marker_`).
   - Для каждого ненулевого элемента $A_{i,k}$ обходится соответствующая строка $k$ матрицы B.
   - Значения суммируются в аккумуляторе: `temp_row[col_b] += val_a * val_b`.
   - Индексы задействованных столбцов сохраняются в `used_cols_`.
2. После обхода всех элементов строки выполняется сортировка `used_cols_` для соблюдения стандарта CRS.
3. Ненулевые значения переносятся в итоговые векторы `values` и `col_ind`.

**Сложность:** $O(NNZ(A) + \text{rows}(A) \times \text{avg\_nnz}(B))$ в среднем.

## 4. Детали реализации
- **Файлы:** `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`.
- **ValidationImpl:** Проверяет размерности и корректность длин `row_ptr`.
- **PreProcessingImpl:** Инициализирует результирующую матрицу и резервирует память под вспомогательные структуры.
- **Фильтрация:** Значения $|x| < 10^{-15}$ игнорируются для численной стабильности.

```cpp
// File: seq/src/ops_seq.cpp
void MaslovaUMultMatrSEQ::ProcessRow(int i, const CRSMatrix &a, const CRSMatrix &b, CRSMatrix &c) {
  used_cols_.clear();
  for (int j = a.row_ptr[i]; j < a.row_ptr[i + 1]; ++j) {
    const int col_a = a.col_ind[j];
    const double val_a = a.values[j];

    for (int k = b.row_ptr[col_a]; k < b.row_ptr[col_a + 1]; ++k) {
      const int col_b = b.col_ind[k];
      const double val_b = b.values[k];

      if (marker_[col_b] < i) {
        marker_[col_b] = i;
        used_cols_.push_back(col_b);
        temp_row_[col_b] = val_a * val_b;
      } else {
        temp_row_[col_b] += val_a * val_b;
      }
    }
  }

  if (!used_cols_.empty()) {
    std::ranges::sort(used_cols_);
    for (int col_idx : used_cols_) {
      const double val = temp_row_[col_idx];
      if (std::abs(val) > 1e-15) {
        c.values.push_back(val);
        c.col_ind.push_back(col_idx);
      }
      temp_row_[col_idx] = 0.0;
    }
  }
}
```

## 5. Проверка корректности
Результаты сравниваются с ожидаемыми значениями в функциональных тестах:
- `FullDense2x2`: проверка на плотных данных.
- `LowerTriangular`: работа с треугольными структурами.
- `OuterProduct` и `VectorInnerProduct`: крайние случаи размерностей.
