## 1. Введение
Задача умножения разреженных матриц является критически важной для научных вычислений и машинного обучения. Основная сложность заключается в неравномерном распределении ненулевых элементов, что делает статическое распределение нагрузки неэффективным. В данной работе исследуется эффективность пяти различных подходов к параллельной реализации этой задачи.

## 2. Единая постановка задачи
Реализовать вычисление результирующей матрицы $C = A \times B$, где все матрицы представлены в формате CRS. 
- **Критерий корректности:** Полное совпадение значений и индексов с последовательным эталоном.
- **Особенности:** Использование двухпроходного алгоритма (символическое вычисление NNZ и заполнение значений) для предотвращения гонок данных без использования блокировок.

## 3. Единая методика эксперимента
Замеры проводились на следующей конфигурации:
- **CPU:** AMD Ryzen 7 7840HS w/ Radeon 780M Graphics.
- **OS:** Windows 11.
- **Сборка:** Release.
- **Параметры:** Случайные матрицы 1000x1000, плотность 5% (50 000 элементов).
- **Агрегация:** Для каждой точки бралась медиана из серии 5 запусков.

## 4. Сводка корректности
Все пять реализаций успешно прошли функциональные тесты:
1. Плотные матрицы (FullDense2x2).
2. Треугольные матрицы (LowerTriangular).
3. Векторные произведения (Inner/Outer Product).
Сравнение с SEQ подтвердило идентичность результатов во всех параллельных режимах.

## 6. Интерпретация различий
1.  **TBB:** Показал наивысшее ускорение (4.99x). Автоматический планировщик и механизм "work-stealing" лучше всего справились с балансировкой нагрузки при обработке строк разной плотности.
2.  **OMP:** Продемонстрировал результат, близкий к TBB. Выбор `schedule(dynamic, 20)` позволил эффективно утилизировать ядра, минимизируя простои.
3.  **STL:** Значительно уступил TBB/OMP. Это обусловлено накладными расходами на ручное создание/уничтожение потоков и отсутствием гибкого перераспределения задач между ними.
4.  **ALL (Hybrid):** В локальном режиме (на одной машине) гибридная версия показала избыточность. Высокие затраты на сборку данных через `MPI_Gatherv` и рассылку `MPI_Bcast` снижают эффективность, однако на 4 рангах время выполнения (0.052с) существенно ниже, чем в однопроцессном режиме.

## 7. Репродуцируемость
Для воспроизведения результатов необходимо:
1. Собрать проект: `cmake -B build -D CMAKE_BUILD_TYPE=Release && cmake --build build --config Release`
2. Запустить тесты производительности:
   - Для потоков: `$env:PPC_NUM_THREADS = 8; ./build/bin/ppc_perf_tests.exe --gtest_filter=*MaslovaU*`
   - Для гибрида: `mpiexec -n 4 ./build/bin/ppc_perf_tests.exe --gtest_filter=*MaslovaU*`

## 8. Заключение
Наиболее эффективной технологией для умножения разреженных матриц в рамках одной системы является **TBB**. Гибридная схема (MPI+OpenMP) рекомендуется для задач, размер которых превышает объем оперативной памяти одного узла, несмотря на высокие накладные расходы на синхронизацию.

## 9. Приложение
Ключевой фрагмент параллельной обработки (двухпроходная схема):
```cpp
// File: omp/src/ops_omp.cpp
#pragma omp parallel default(none) shared(a, b, c, rows_a, cols_b)
{
  std::vector<double> acc(cols_b, 0.0);
  std::vector<int> marker(cols_b, -1);
  std::vector<int> used;
#pragma omp for schedule(dynamic, 20)
  for (int i = 0; i < rows_a; ++i) {
    FillRowValues(i, a, b, c, acc, marker, used);
  }
}
```
