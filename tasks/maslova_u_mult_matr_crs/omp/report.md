## 1. Контекст
Умножение разреженных матриц характеризуется неравномерной вычислительной нагрузкой на разные строки. Технология OpenMP позволяет распределить обработку строк между ядрами процессора, используя динамическую балансировку.

## 2. Постановка задачи
Параллельная реализация алгоритма умножения матриц A и B в формате CRS с использованием директив OpenMP. Основная цель — сокращение времени выполнения на многоядерных системах. Сравнение проводится с [SEQ](../seq/report.md).

## 3. Базовый алгоритм
Используется двухпроходная схема для исключения состояния гонки (race condition) и избыточных аллокаций:
1. **Символический проход:** Вычисление количества ненулевых элементов (NNZ) для каждой строки результата.
2. **Вычислительный проход:** Непосредственное вычисление значений и заполнение заранее выделенных массивов.

## 4. Схема распараллеливания
Параллелизм реализован на уровне строк матрицы A:
- **Директива:** `#pragma omp parallel`.
- **Область данных:** 
    - `shared`: исходные матрицы A, B и результирующая C.
    - `private`: для каждого потока создаются свои локальные буферы `marker`, `acc` и `used`.
- **Планировщик:** `schedule(dynamic, 20)`. Выбран динамический тип, так как строки имеют разное количество элементов, что приводит к дисбалансу при статическом делении.

```cpp
// File: omp/src/ops_omp.cpp
#pragma omp parallel default(none) shared(a, b, c, rows_a, cols_b) num_threads(ppc::util::GetNumThreads())
{
  std::vector<double> acc(cols_b, 0.0);
  std::vector<int> marker(cols_b, -1);
  std::vector<int> used;
  used.reserve(cols_b);

#pragma omp for schedule(dynamic, 20)
  for (int i = 0; i < rows_a; ++i) {
    FillRowValues(i, a, b, c, acc, marker, used);
  }
}
```

## 5. Детали реализации
- **Файлы:** `omp/include/ops_omp.hpp`, `omp/src/ops_omp.cpp`.
- **Синхронизация:** Благодаря двухпроходному алгоритму потоки пишут в разные участки массивов `values` и `col_ind`, что исключает необходимость в `critical` секциях или `mutex`.
- **Память:** Результирующие векторы `resize`-ятся один раз между проходами.

## 6. Проверка корректности
Корректность подтверждена прохождением 5 функциональных тестов. Результат OMP-версии полностью совпадает с SEQ-версией на всех наборах данных.
