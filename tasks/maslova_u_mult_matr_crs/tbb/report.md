## 1. Контекст
Использование TBB позволяет абстрагироваться от ручного управления потоками и использовать масштабируемую систему задач. Это особенно эффективно для умножения разреженных матриц, где объем вычислений для разных строк непредсказуем.

## 2. Постановка задачи
Реализовать параллельное умножение CRS-матриц с использованием примитивов TBB. Сравнить производительность с последовательным эталоном [SEQ](../seq/report.md).

## 3. Базовый алгоритм
Алгоритм основан на двухпроходной схеме (символический и вычислительный этапы). Это позволяет избежать блокировок при записи в общую память, так как каждый поток работает со своим заранее определенным диапазоном индексов в результирующих векторах.

## 4. Схема распараллеливания
- **Примитив:** `tbb::parallel_for`.
- **Разбиение:** Использован `tbb::blocked_range<int>(0, rows_a)`, который автоматически делит матрицу на блоки строк (chunks).
- **Локальность данных:** Внутри лямбда-выражения для каждого диапазона создаются локальные объекты `marker`, `acc` и `used`. Это минимизирует конфликты кэша и обеспечивает потокобезопасность.
- **Планировщик:** TBB использует алгоритм кражи задач (work-stealing), что обеспечивает автоматическую балансировку нагрузки между ядрами.

## 5. Детали реализации
- **Файлы:** `tbb/include/ops_tbb.hpp`, `tbb/src/ops_tbb.cpp`.
- **Методы:** `GetRowNNZ` (подсчет элементов) и `FillRowValues` (вычисление).
- **Особенности:** Результирующие массивы `values` и `col_ind` изменяют размер ровно один раз между двумя вызовами `parallel_for`.

```cpp
// File: tbb/src/ops_tbb.cpp
tbb::parallel_for(tbb::blocked_range<int>(0, a.rows), [&](const tbb::blocked_range<int> &r) {
  std::vector<double> acc(b.cols, 0.0);
  std::vector<int> marker(b.cols, -1);
  std::vector<int> used;
  used.reserve(b.cols);
  for (int i = r.begin(); i < r.end(); ++i) {
    FillRowValues(i, a, b, c, acc, marker, used);
  }
});
```
